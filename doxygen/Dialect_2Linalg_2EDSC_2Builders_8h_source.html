33 /// Build a `linalg.generic` op with the specified `inputs`, `outputs` and
34 /// `region`.
35 ///
36 /// `otherValues` and `otherAttributes` may be passed and will be appended as
37 /// operands and attributes respectively.
38 ///
39 /// Prerequisites:
40 /// =============
41 ///
42 /// 1. `inputs` may contain StructuredIndexed that capture either buffer or
43 /// tensor values.
44 /// 2. `outputs` may contain StructuredIndexed that capture either buffer values
45 /// or tensor types. If both buffer values and tensor types are present, then
46 /// all buffer values must appear before any tensor type. Without this
47 /// restriction output tensor results would need to be reordered, which would
48 /// result in surprising behavior when combined with region definition.
49 Operation *makeGenericLinalgOp(
50  ArrayRef<IteratorType> iteratorTypes, ArrayRef<StructuredIndexed> inputs,
51  ArrayRef<StructuredIndexed> outputs,
52  function_ref<void(ValueRange)> regionBuilder = defaultRegionBuilder,
53  ArrayRef<Value> otherValues = {}, ArrayRef<Attribute> otherAttributes = {});

74 /// Build a linalg.pointwise, under the current ScopedContext, at the current
75 /// insert point, that computes:
76 /// ```
77 /// (i0, ..., in) = (par, ..., par)
78 /// |
79 /// | O...(some_subset...(i0, ..., in)) =
80 /// | some_pointwise_func...(I...(some_other_subset...(i0, ..., in)))
81 /// ```
82 ///
83 /// This is a very generic entry point that can be configured in many ways to
84 /// build a perfect loop nest of parallel loops with arbitrarily complex
85 /// innermost loop code and whatever (explicit) broadcast semantics.
86 ///
87 /// This can be used with both out-of-place and in-place semantics.
88 /// The client is responsible for ensuring the region operations are compatible
89 /// with in-place semantics and parallelism.
90 
91 /// Unary pointwise operation (with broadcast) entry point.
92 using UnaryPointwiseOpBuilder = function_ref<Value(Value)>;
93 Operation *linalg_generic_pointwise(UnaryPointwiseOpBuilder unaryOp,
94  StructuredIndexed I, StructuredIndexed O);
