28 /// FuncOp represents a function, or an operation containing one region that
29 /// forms a CFG(Control Flow Graph). The region of a function is not allowed to
30 /// implicitly capture global values, and all external references must use
31 /// Function arguments or attributes that establish a symbolic connection(e.g.
32 /// symbols referenced by name via a string attribute).
