188 // A class for maintaining information for symbols bound in patterns and
189 // provides methods for resolving them according to specific use cases.
190 //
191 // Symbols can be bound to
192 //
193 // * Op arguments and op results in the source pattern and
194 // * Op results in result patterns.
195 //
196 // Symbols can be referenced in result patterns and additional constraints to
197 // the pattern.
198 //
199 // For example, in
200 //
201 // ```
202 // def : Pattern<
203 // (SrcOp:$results1 $arg0, %arg1),
204 // [(ResOp1:$results2), (ResOp2 $results2 (ResOp3 $arg0, $arg1))]>;
205 // ```
206 //
207 // `$argN` is bound to the `SrcOp`'s N-th argument. `$results1` is bound to
208 // `SrcOp`. `$results2` is bound to `ResOp1`. $result2 is referenced to build
209 // `ResOp2`. `$arg0` and `$arg1` are referenced to build `ResOp3`.
210 //
211 // If a symbol binds to a multi-result op and it does not have the `__N`
212 // suffix, the symbol is expanded to represent all results generated by the
213 // multi-result op. If the symbol has a `__N` suffix, then it will expand to
214 // only the N-th *static* result as declared in ODS, and that can still
215 // corresponds to multiple *dynamic* values if the N-th *static* result is
216 // variadic.
217 //
218 // This class keeps track of such symbols and resolves them into their bound
219 // values in a suitable way.
220 class SymbolInfoMap {
221 public:
222  explicit SymbolInfoMap(ArrayRef<llvm::SMLoc> loc) : loc(loc) {}
